= 1. General overview of Mule

== What is MuleSoft and which problem does it solve

MuleSoft offers one of the most widespread integration platforms for connecting SaaS and enterprise applications in cloud and on-premise environments.
In other words, MuleSoft is a platform that gives IT departments tools for automatic system and data integration, accelerating workflows, and creating digital experiences – all on a unified, simple platform.

In modern enterprises, there are many different systems (ERP, CRM, databases, cloud services, internal and external applications) that need to exchange data.
MuleSoft solves the problem of connecting these diverse systems by providing a scalable, secure, and maintainable integration platform.

== What is Mule 4

At the core of the MuleSoft platform lies the *Mule 4 runtime* – an integration engine and development platform for building integration applications and APIs.
Mule 4 is written in Java and is based on an *event-driven architecture (EDA)*; it is a lightweight and scalable engine that automatically manages thread and event execution.

For the developer, this means that the focus can shift from low-level details (such as threads, sockets, and memory) to designing integration flows (*flows*) and business logic.

Technically speaking, the Mule runtime uses proven Java technologies in the background (e.g., it is built on the Spring framework), but the developer does not write a classic Java application.
Mule provides a higher level of abstraction through visual composition of integration components.
The goal is to build integrations declaratively and faster, with minimal so-called *"glue" code*.

Mule 4 represents an evolution of the Mule 3 runtime.
It retains the same key elements (applications, flows, connectors, DataWeave, etc.), but introduces a simpler message model and a richer set of capabilities, so there are fewer concepts to learn and maintain.

== Development tools and platform

The official development environment for Mule is *Anypoint Studio* – an Eclipse-based IDE, adapted for visually designing Mule applications.

In Anypoint Studio:

* we create Mule projects,
* add connectors from a rich palette (over 200 ready-made connectors for systems like Salesforce, SAP, Workday, databases, and others),
* develop flows by drag-and-drop method and component configuration.

Studio comes with all necessary runtime elements (embedded Mule server, pre-installed JDK and Maven) and enables local testing and debugging of applications.

MuleSoft has also introduced *Anypoint Code Builder* – a new generation IDE based on the Visual Studio Code platform.
Code Builder provides the same functionality as Studio, but is available:

* as a VS Code extension (desktop),
* as a cloud IDE inside the Anypoint Platform.

This enables developing Mule applications directly in the browser with connection to the Anypoint account, which brings additional flexibility, especially for team collaboration and working on different operating systems.

Besides local development, MuleSoft provides a central platform for deploying and managing applications – *Anypoint Platform*.

Within the platform, *CloudHub* represents the cloud environment (MuleSoft’s iPaaS) where Mule applications can be deployed with one click or scripted.

A typical development cycle looks like this:

. local development and testing in Anypoint Studio,
. deployment of the application to CloudHub (sandbox or production),
. application management through Anypoint Platform tools (e.g., Runtime Manager).

Studio is directly integrated with Anypoint Platform, enabling deployment of applications to the cloud directly from the IDE.
Local development and cloud execution are not separate processes, but part of a unified development cycle.

== MuleSoft community and certification

Formal certification is not mandatory to start working with MuleSoft.
However, the *MuleSoft Certified Developer (MCD) Level 1* is a good indicator of the basic knowledge expected from a Mule developer.

It is recommended to use the domains covered by this certification as a guide through the curriculum, including:

* working in Anypoint Studio,
* development of flows,
* usage of connectors,
* basic data transformations,
* error handling.

The certificate serves as a knowledge verification, but learning the technology can be started without it.
